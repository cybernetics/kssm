me.angrybyte.kssm.api.Action

A fundamental building block of a StateMachine. In this example that would be A-to-B, B-to-C, C to A and C : Self.

me.angrybyte.kssm.api.ErrorHandler

A suspending shorthand for reacting to Throwables coming from a StateMachine.

me.angrybyte.kssm.api.data.IncubationException

Signals the catcher that StateMachine is in incubation state. Used to replace null in the observable flow of errors.

me.angrybyte.kssm.api.data.IncubationPassedException

Signals the catcher that StateMachine has passed the incubation state. No more configuration changes should be made on the state machine instance.

me.angrybyte.kssm.api.data.NoMappingException

Signals the catcher that StateMachine doesn't have the mapping registered for the given state and action. The easiest fix is to check your mappings and add the missing ones.

me.angrybyte.kssm.api.StateMachine

The public-facing API for configuring, using and observing changes within a finite state machine.

me.angrybyte.kssm.dsl.StateMachineDsl

The DSL (domain-specific language) wrapper. Allows easy state machine configuration and instantiation.

me.angrybyte.kssm.api.data.StateMapping

Keeps information about a particular state-to-state transition mapping. You should provide an action at creation time, and both start and end states surrounding the action.

me.angrybyte.kssm.dsl.StateVector

Holds a State and an Action, without knowing anything about the resulting state.

In most ways a structurally similar data model to StateMapping. In contrast to StateMapping, a Transition is mostly used to report already processed state changes to the outside world.
